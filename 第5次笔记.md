# 单片机通信方式
## 双向串口通信
有两根通信线（发送端TXD和接收端RXD）交叉连接。单向数据传输可以只需要一根传输线。  
当电平标准不一致时，需要加电平转换芯片。  
【电平标准】：数据1和0的表示方式，是人为规定的电压与数据的关系。（串口常用的电平标准;TTL、RS232、RS485）  
【差分信号】：消除同频干扰信号，可使通讯距离延长。  
## 常见通信接口
| 名称    | 引脚定义                 | 通信方式       | 特点             |
|---------|--------------------------|----------------|------------------|
| UART    | TXD、RXD                 | 全双工、异步   | 点对点通信       |
| I2C_I   | SCL（时钟线）、SDA                 | 半双工、同步   | 可挂载多个设备   |
| SPI     | SCLK、MOSI、MISO、CS     | 全双工、同步   | 可挂载多个设备   |
| 1-Wire  | DQ                       | 半双工、异步   | 可挂载多个设备   |
## 通信方式
| 术语       | 定义                                                         |
|------------|--------------------------------------------------------------|
| 全双工     | 通信双方可以在同一时刻互相传输数据                           |
| 半双工     | 通信双方可以互相传输数据，但必须分时复用一根数据线           |
| 单工       | 通信只能有一方发送到另一方，不能反向传输                     |
| 异步       | 通信双方各自约定通信速率                                     |
| 同步       | 通信双方靠一根时钟线来约定通信速率                           |
| 总线       | 连接各个设备的数据传输线路（类似于一条马路，把路边各住户连接起来，使住户可以相互交流） |
## 51单片机UART寄存器配置
STC89C52中只有一个UART，有四种工作模式，其中最常用的是模式1（8位UART，波特率可变）。  
TH1、TL1：定时器1的初值寄存器。在模式2下，TH1存放自动重装的初值，TL1是计数器  
SBUF：串口数据缓存寄存器，两个独立的寄存器占用相同地址。
### 特殊功能寄存器
| 寄存器     | 关键位       | 功能描述                                                                 |
|------------|--------------|--------------------------------------------------------------------------|
| SCON       | SM0、SM1     | 串行口工作方式选择位。设置 SM0=0, SM1=1 即选择模式1（8位UART）。        |
| SCON       | REN          | 接收允许位。软件置 REN=1 时，允许串口接收数据。                          |
| SCON       | TI           | 发送中断标志。发送完一帧数据后，硬件自动置1。需软件清零。                |
| SCON       | RI           | 接收中断标志。接收完一帧数据后，硬件自动置1。需软件清零。                |
| PCON       | SMOD         | 波特率选择位。SMOD=1 时，波特率加倍；SMOD=0 时，波特率不加倍。           |
| TMOD       | -            | 定时器模式寄存器。我们需要配置定时器1工作在模式2，即8位自动重装模式，作为波特率发生器。 |
|
### 串口参数
波特率：串口通信的速率（发送和接收各数据位的间隔时间）  
校验位：用于数据验证。eg.奇偶校验  
### 配置方式
【1】设置定时器1为波特率发生器，由于其波特率由定时器1的溢出率决定，通常将定时器1配置为模式2；  
【2】装载定时器初值；  
【3】配置串口工作模式：设置SCON寄存器，选择模式1并允许接收；  
### 串口初始化函数（9600波特率）
void UART_Init(void)  
{
	PCON &= 0x7F;     
	SCON = 0x50;  			
	TMOD &= 0x0F;		
	TMOD |= 0x20;		
	TL1 = 0xFD;		
	TH1 = 0xFD;		
	ET1 = 0;		
	TR1 = 1;		
}
## 串口的收发函数
### 发送函数
将数据放入函数，等待T1置1，然后将其清零  
void UART_SendByte(unsigned char byte)  
{
    SBUF = byte;  
    while (TI == 0);   // 等待发送完成  
    TI = 0;            // 清零标志  
}
### 接收函数
在串口初始化时需开启中断：EA=1（总中断）;ES=1（串口中断）;  
unsigned char rxData = 0;           // 存放接收到的数据  
bit rxFlag = 0;                     // 接收完成标志，在主循环中rxFlag如果为1，表示接收到了新的数据。  


void UART_ISR() interrupt 4 {  
    if (RI == 1) {                   // 如果是接收中断  
        RI = 0;                      // 软件清零RI  
        rxData = SBUF;                // 读取数据  
        rxFlag = 1;                   // 设置接收标志  
    }  
    // 如果是发送中断，我们不用中断发送，所以可以忽略TI  
    // 如果使用了发送中断，这里也要处理TI  
}
